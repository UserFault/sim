Описание механизма получения списка поддерживаемых типов БД.
Версия Тапп Бар 1.0.0.0

Доступ к БД выполняется через класс АдаптерБД.
Для каждого типа БД и СолюшенБезБД приходится писать собственный класс АдаптерБД.

Енум MDatabaseType описывает все типы БД, поддерживаемые движком в текущей версии.
С помощью него можно указать тип БД при создании Солюшена.
Также, при открытии Солюшена проверяется, поддерживается ли БД типа, указанного в файле настроек Солюшена.

Адаптеры БД организованы в иерархию классов.
Базовым классом в этой иерархии является класс BaseDbAdapter.
Он лишь задает интерфейс для производных классов, которые и выполняют работу.

Каждый класс адаптера БД должен иметь атрибут MDatabaseTypeAttribute.
Этот атрибут имеет два параметра:
 - MDatabaseType тип поддерживаемой БД
 - bool Active - флаг что данный адаптер закончен, отлажен и готов для работы в Движке.

Например, класс  BaseDbAdapter имеет атрибут:
  [MDatabaseTypeAttribute(MDatabaseType.Unknown, false)]
 Это значит, что тип БД - MDatabaseType.Unknown и адаптер нельзя использовать для работы.

Для каждого типа БД может быть активен только один адаптер БД.
Для каждого адаптера БД можно указать только один тип БД.
Иначе Движок будет выбрасывать исключения.

Поиск адаптеров производится с помощью Reflection в сборке Движка.
В других сборках поиск не проводится. 
Поэтому все адаптеры БД должны размещаться в сборке собственно Движка.

Сами функции получения списка поддерживаемых типов БД вынесены в класс MUtility.
Пока их негде разместить в другом месте.
- public static bool IsSupportedDbType(MDatabaseType dbType) - проверить, поддерживается ли укаанный тип БД
- public static List<MDatabaseType> GetAvailableDatabaseTypes() - получить сисок поддерживаемых типов БД.
