Я не знаю, какой способ лучше выбрать. 
Текущий проще в плане вызовов методов для ячеек.
Но пользователь работает все же с солюшеном.
- В первом варианте функции солюшена в корневом объекте, а функции ячеек в вложенном.
- В текущем же варианте - наоборот, зато меньше функций тянуть из контейнера в первый слой.
- выбрал пока первый вариант. Функции движка буду брать из объекта контейнера.

Устройство Контейнера:

- Контейнер
  - МенеджерИдЯчеек
  - МенеджерИдСвязей
  - КоллекцияЯчеек
  - КоллекцияСвязей
  - АдаптерБД
  - НастройкиСолюшена
  - группа свойств Контейнера по интерфейсу
  - ImSerializable


АдаптерБД:
- BaseDbAdapter - задает интерфейс адаптера БД, не содержит собственно кода (уточнить)
  - MsJetDbAdapter - адаптер БД Аксцесс
  - NoDbAdapter - адаптер для варианта без БД

Для этой части еще потребуется енум MDatabaseType для НастройкиСолюшена.
Но его надо дополнять значениями толко при наличии самих адаптеров.
А то будет плохо - выбирать несуществующий адаптер БД.
И можно в BaseDbAdapter предусмотреть поле ТипАдаптера, 
  который значение из этого енума будет возвращать.
  Это для автоматизации определения доступных адаптеров.
  Тогда сразу можно будет определить, При создании нового Солюшена, поддерживается ли этот тип БД Движком или нет.
