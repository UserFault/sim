Тестовая оболочка

Описание

Тестовая оболочка предназначена для отработки решений визуализации структуры проекта, создания модели пользовательской концепции проекта. Она содержит средства для управления проектом, структурой данных проекта.

Изображения и описания приложения могут немного отличаться от самого приложения в связи с его доработкой.

Проект создается в соответствии с пользовательской концепцией проекта, которая является надстройкой общей концепции проекта, что определяет средства и ограничения проекта.

Общая концепция проекта

Проект представляет собой контейнер, в котором существуют ячейки и связи. Ячейки являются сущностями пользовательской концепции проекта, а связи являются отношениями между этими сущностями. Общая концепция оперирует ячейками и связями в контейнере, предоставляя пользователю некоторый базовый набор операций с минимумом ограничений. Используя этот набор, пользователь разрабатывает и создает проект в некоторой предметной области.

Пользовательская концепция проекта

Пользовательская концепция определяет структуру ячеек и связей, использование свойств ячеек и связей, операции над структурой и методику разработки, создания и выполнения этих операций. Пользовательская концепция ограничена общей концепцией проекта как фундаментом.

Операции

Операции с проектом

Создать новый проект

Создание нового пустого проекта. Меню Project-New... Появится диалог создания нового проекта.


  1. Введите имя создаваемого проекта. Имя проекта не должно быть длинным, не должно содержать пробелов.
  2. Введите описание создаваемого проекта.
  3. Укажите родительский каталог для проекта. У пользователя должно быть достаточно прав для создания каталогов и файлов проекта. Типично это папка Мои документы.
  4. Если проект будет использовать базу данных, установите флажок. Проект будет использовать базу данных Microsoft Sql Server 2005. Проект с базой данных хранит все данные в базе.
  5. Если проект не будет использовать базу данных, снимите флажок. Проект без базы данных во время работы располагает все данные в оперативной памяти, а затем они должны быть сохранены в файл полного снимка проекта. Работа проекта без базы данных еще не тестировалась.
  6. Для использования базы данных укажите путь к серверу баз данных. Если это поле пустое, считается что проект не использует базу данных.
  7. Введите имя новой базы данных. Если оставить поле пустым, имя будет создано из имени проекта.
  8. Введите логин пользователя сервера баз данных. Пользователь должен быть членом серверной роли dbcreator, то есть, иметь право создавать базы данных. Сервер должен быть настроен на смешанную авторизацию, то есть поддерживать авторизацию по логину и паролю.
  9. Введите пароль пользователя. Сейчас для упрощения процесса отладки этот пароль хранится в файле проекта в открытом виде, что небезопасно.
  10. Укажите таймаут соединения с сервером в секундах, если вас не устраивает значени по умолчанию.
  11. Нажмите кнопку Создать. Будет создан новый пустой проект. Для работы с ним надо открыть проект.

Открыть существующий проект

Меню Project-Open... открывает существующий проект. В диалоге открытия файла надо перейти в каталог нужного проекта и выбрать файл проекта. Будет открыт проект.

Закрыть проект

Меню Project-Close закрывает открытый проект. Перед закрытием предлагается сохранить проект. При сохранении проекта, использующего базу данных, в базу данных сохраняются ячейки отложенной записи.

Сохранение проекта, не использующего базу данных, не имеет смысла. Для сохранения такого проекта необходимо создать полный снимок проекта.

Очистить проект

Меню Project-Clear позволяет удалить из проекта все ячейки и связи, из памяти и базы данных. Файлы ресурсов, снимки, логи, сборки методов не удаляются.

Удалить проект

Меню Project-Delete... позволяет удалить проект с диска и сервера баз данных. Если на компьютере работает служба индексирования или восстановления, они могут блокировать некоторые файлы проекта, при этом операция дает сбой.

Выход из приложения

Меню Project-Exit позволяет закрыть приложение. Закрыт ли проект,  не проверяется.

Операции со снимками

Снимок структуры хранит полную копию структуры данных. Этот снимок может быть создан или загружен. В течение работы можно создать несколько снимков для сохранения промежуточных состояний структуры данных.

Создать полный снимок

Меню Snapshot-Create snapshot позволяет создать полный снимок структуры.  В течение работы можно создать несколько снимков для сохранения промежуточных состояний структуры данных. Снимки являются версиями структуры данных, их можно использовать для отката изменений.

Загрузить полный снимок

Меню Snapshot-Load snapshot позволяет загрузить полный снимок структуры в открытый проект. Загружать снимок следует в пустой или очищенный проект.

Навигатор

Навигатор позволяет перемещаться по структуре данных проекта, редактировать ячейки и связи.

Форма навигатора

Форма навигатора представляет структуру данных проекта. Пользователь перемещается от ячейки к ячейке по связям между ячейками.

В центральной части формы находится окно, представляющее  текущую ячейку. В верхней части формы находится окно, в котором отображаются над-ячейки, связанные с текущей ячейкой. В нижней части формы находится окно, в котором отображаются под-ячейки, связанные с текущей ячейкой. Смысл расположения над/под определяется смыслом связей.

Если у ячейки много связанных ячеек, их отображение может занимать длительное время.

Ячейки в представлениях группируются по осям связей. Ось представляет собой некоторый смысл отношений между ячейками. Название оси показывается подчеркнутой надписью. Одна и та же ячейка может несколько раз присутствовать  в представлении, если имеет несколько связей с текущей ячейкой.

Неактивные связи представлены серым цветом. Активные связи изображены черным цветом.

 

Операции формы навигатора

Меню View позволяет выбрать способ отображения связанных ячеек.

Создание ячейки

Меню Actions-Cell-Create... позволяет создать ячейку. Создается новая ячейка. Появляется форма ячейки, в которой надо указать  параметры этой новой ячейки. При нажатии на кнопку ОК форма ячейки закрывается и изменения записываются в ячейку. Если нажать на кнопку Отмена формы ячейки, новая ячейка помечается неактивной, но не удаляется из структуры.

Редактирование свойств ячейки

Меню Actions-Cell-Edit... позволяет просмотреть и изменить свойства ячейки. Появляется форма ячейки, в которой можно посмотреть и изменить  параметры  ячейки. При нажатии на кнопку ОК форма ячейки закрывается и изменения записываются в ячейку.

Удаление ячейки

Для удаления ячейки необходимо сделать ее неактивной при помощи операции редактирования свойств ячейки. Для реального удаления ячейки необходимо выполнить операцию оптимизации проекта. Она не реализована.

Создание связи

Меню Actions-Link-Create... позволяет создать связь. Создается новая пустая связь. Появляется форма связи, в которой надо указать  параметры содаваемой связи. При нажатии на кнопку ОК форма связи закрывается и изменения записываются в пустую связь. Затем между соответствующими ячейками создается настоящая связь, и ей присваиваются указанные параметры. Если нажать на кнопку Отмена формы связи, связь не создается.

 Редактирование свойств связи

Контекстное меню представлений связанных ячеек позволяет просмотреть и изменить свойства связи. При клике правой кнопкой мыши на некоторой ячейке, появляется контекстное меню, которое содержит текстовые описания связей, существующих  между выбранной и текущей ячейками. При выборе пункта появляется форма связи, в которой можно посмотреть и изменить  параметры  связи. Нельзя изменять ячейки связи. При нажатии на кнопку ОК форма связи закрывается и изменения записываются в связь. При нажатии на кнопку Отмена изменения не сохраняются.

Удаление связи

Для удаления связи необходимо сделать ее неактивной при помощи операции редактирования свойств связи. Для реального удаления связи необходимо выполнить операцию оптимизации проекта. Она не реализована.

Редактирование свойств контейнера

Меню Actions- Container-Edit позволяет редактировать свойства контейнера. Форма контейнера  не реализована.

Возврат в предыдущее окно

Меню Actions-Return current cell позволяет вернуться в предыдущее окно и принести с собой текущую ячейку. Этот механизм используется для указания идентификаторов ячеек в формах. При этом попутно можно выполнять любые другие операции. Пользователь должен самостоятельно учитывать последствия возможной рекурсии.

Планируется также добавить возврат нескольких ячеек, но пока такая возможность не реализована.

Форма ячейки

Форма ячейки представляет ячейку.

Форма предназначена для просмотра и редактирования данных ячейки. В полях, где необходимо указывать идентификаторы ячеек, есть кнопки. При нажатии на такую кнопку открывается новое окно навигатора, в котором  пользователь может произвести любые операции, выбрать ячейку и вернуться с ней в предыдущее окно. Вместо идентификатора ячейки в поле отображается имя ячейки.

Кнопка рядом с полем для имени ячейки позволяет проверить уникальность имени ячейки в проекте. Для ячеек, имена которых должны быть уникальными.

Флажок активности обозначает состояние активности ячейки. Если он снят, ячейка считается удаленной. Удаление производит операция оптимизации проекта. Она пока не готова.

 Форма связи представляет связь.

Форма предназначена для просмотра и редактирования данных связи. В полях, где необходимо указывать идентификаторы ячеек, есть кнопки. При нажатии на такую кнопку открывается новое окно навигатора, в котором  пользователь может произвести любые операции, выбрать ячейку и вернуться с ней в предыдущее окно.

Флажок активности обозначает состояние активности связи. Если он снят, связь считается удаленной. Удаление производит операция оптимизации проекта. Она пока не готова.
